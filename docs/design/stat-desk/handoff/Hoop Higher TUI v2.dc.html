01 · Direction — STAT DESK

Keeping the widgets, fixing the look

You liked the real widget structure — so it stays. Four targeted fixes on top of it:

One accent

Blue-everywhere → warm orange for focus/brand only. Data stays neutral so the eye goes where it should.

Colour = meaning

Buttons aren't green/red by default (that read as pre-judged). Green/red appear only on the reveal.

Points as hero

Empty panels → the point total is the biggest thing on the card; name/team/min are agate around it.

Less chrome

Dropped the redundant centred title/score band. One scorebug row carries mode, round, Q, score, streak.

02 · Color system — light + dark

Two palettes, one set of roles

Dark ships default; light is a full sibling. Widgets reference a role — accent, success, danger… — never a raw hex, so a theme swap is one variable table. Each maps to a 256/16 fallback that keeps correct green, wrong red, focus orange.

04 · Responsive — how the Game screen adapts

One screen, four terminal sizes

Detail sheds in a fixed priority order as space shrinks; the matchup and the two guess actions never leave. Breakpoints match the app: lg ≥128 · md 96–127 · sm 72–95 · xs <72.

05 · Light mode

The same widgets on paper

Identical layout, roles remapped: borders warm-tan, text ink, orange deepened to #E0521F to hold contrast on paper. Toggled from a setting or terminal theme; choice persists per machine.

06 · Interaction spec

How it feels to play

Focus

The focused widget fills solid orange (paper: #E0521F); everything else is a hairline outline with a muted label. ←/→ toggle the two guess buttons, ↑/↓ walk menus. Focus is a fill change, never colour-only — legible on 16-colour terminals via reverse video.

The 1.2s reveal

On guess both buttons dim to disabled instantly (no double-submit). Player B's — — PTS flips to the real number and its whole panel border + number turn green (right) or red (wrong) with a ▲/▼. A verdict strip drops in with the trash-talk line + signed delta; the scorebug score recolours for one beat. After 1.2s the next question swaps in — or a modal takes over.

Loading & errors

Picking a mode disables all three cards and drops an amber status strip with a braille spinner. Message escalates on a 1s tick: <5s "Loading…" → 5–15s "Still fetching NBA games (n s)…" → >15s "stats.nba.com is slow; esc to cancel." Back becomes CANCEL. Failure raises the red "Unable to start game" toast and returns to idle.

The scorebug

One row pinned to the top of Game: brand + mode + round + Q on the left, score + streak + best on the right. It replaces the old redundant centred title band, and is the single constant that carries run status through every question and modal.

Degradation order

As width drops, things leave in this order: games strip detail → last-guess line → side-by-side layout (players stack) → button labels (HIGHER→HI). The matchup and both actions are never dropped. Short heights let the game body scroll while scorebug + buttons + footer stay pinned.

Keys & discovery

Every screen ends in a dim footer of live bindings (H L · ←/→ · 1 2 3 · enter · esc · Q), so muscle-memory keys are always one glance away. Letter shortcuts sit bracketed inside their own labels — binding and target never separated. No action is mouse-only.
